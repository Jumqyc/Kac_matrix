# Virasoro Kac Matrix

A Python script that computes the **Kac matrix** (Gram matrix) of the
Verma module for the Virasoro algebra at a given level.  The entries are
symbolic polynomials in the central charge `c` and the highest weight `h`.

## Mathematical background

The **Virasoro algebra** is the infinite‑dimensional Lie algebra
generated by `L_n` (n ∈ ℤ) with commutation relations $$[L_m, L_n] = (m - n) L_{m+n} + c/12 (m³ - m) δ_{m+n,0}$$

A **highest‑weight representation** is built on a state `|h⟩` satisfying
`L_0|h⟩ = h|h⟩` and `L_n|h⟩ = 0` for `n > 0`.  The states of the Verma module
at integer level `N` are spanned by $$L_{-λ₁} L_{-λ₂} … L_{-λ_k} |h⟩ $$ 
where `λ = (λ₁, λ₂, …)` is a partition of `N` into positive integers,
usually written in non‑increasing order.

The **Kac matrix** (or Gram matrix) `M(N)` records the inner products
between these basis states.  Its determinant, the **Kac determinant**,
gives the condition for the existence of singular (null) vectors.  The
matrix elements are computed recursively using the algebra commutation
relations.

## What the script does

- Generates all integer partitions of the given level `N`.
- For every pair of partitions, builds the appropriate product of
  Virasoro generators and evaluates the vacuum expectation value
  `⟨h| … |h⟩` symbolically.
- Returns the symmetric matrix of inner products as SymPy expressions.
- Caches intermediate results (`functools.lru_cache`) for speed.
- Prints the matrix in both SymPy pretty‑print and a plain‑text format
  (with `^` for exponentiation and no `*` for multiplication).

## Dependencies

- Python ≥ 3.8
- [SymPy](https://www.sympy.org/) (install with `pip install sympy`)

The script uses only the standard library and SymPy; NumPy is **not**
required.

## Usage

Run from the command line:

```bash
python kac.py [level]
```

If no level is given, the default is 4.

## Example:

```python
kac.py 3
Kac matrix for level 3:
⎡         2                          ⎤
⎢ 2⋅c + 8⋅h  + 8⋅h         6⋅h    6⋅h⎥
⎢                                    ⎥
⎢                          2         ⎥
⎢    6⋅h          4⋅h + 8⋅h     0    ⎥
⎢                                    ⎥
⎢                                3   ⎥
⎣    6⋅h              0        24⋅h   ⎦
```

## Performance
The number of partitions grows rapidly (≈ p(N) ~ e^{π√(2N/3)} / 4N√3),
so computation becomes expensive for N > 8–9. Memoization greatly
reduces the number of recursive calls, but the exponential complexity
remains. For higher levels you may need to add more optimisations or use
a dedicated determinant formula.
